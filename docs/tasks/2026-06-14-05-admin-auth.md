# Task 05: Admin Auth For External Bind

## Goal

The admin UI must be safe to bind outside localhost by requiring authentication
when external access is enabled.

## Scope

- Add admin auth config with local-default relaxed behavior and external-bind
  protection.
- Support a simple local token/password flow suitable for the MVP.
- Gate API and UI routes consistently.
- Keep localhost development ergonomic.
- Add smoke/test coverage for unauthenticated and authenticated requests.

## Definition Of Done

- Binding to `0.0.0.0` without auth gives a clear blocked/degraded diagnostic or
  requires explicit opt-in.
- Protected routes reject missing/invalid auth when protection is enabled.
- Localhost default remains easy for development.
- Tests cover both paths.

## Progress

- [x] Scope captured.
- [x] Implementation completed.
- [x] Tests run: `cargo test --quiet`.
- [x] Roadmap updated.
- [x] Committed separately: `b2d5283 Require auth for external admin binds`.

## Result

Admin config now supports `auth_enabled` and `auth_token`, with
`LIBRARIAN_ADMIN_TOKEN`/`LIBRARIAN_ADMIN_AUTH` overrides. Localhost remains
frictionless by default, but externally reachable binds are blocked unless auth
is enabled and a token is configured. Protected admin routes accept
`Authorization: Bearer <token>` or `?admin_token=<token>`.
